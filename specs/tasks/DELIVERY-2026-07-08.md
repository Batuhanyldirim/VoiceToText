## 1. Markdown-rendered notes  (`3f67155`)
The clinical note was stored as Markdown but shown as raw monospace source
(`#`, `**`, `-` visible). Added a small self-contained `Markdown.tsx` renderer
(styled MUI headings/lists/bold/quotes/code; **no new dependency**; renders text
as React children → XSS-safe for PHI). The finished note + the "Klinik İnceleme
Gerekli" card now render formatted; live streaming still shows raw text + cursor
until done.

## 2. Editable & finalizable notes — Tier 1  (`ea40756`, REQ-132–136, ADR-0015)
- **Edit** the note (markdown editor, Kaydet/İptal). Edits are an **overlay** —
  the AI original is never overwritten, so there's an audit trail + safe revert.
- **Tamamla/İmzala** → locks the note as a final record (draft warning turns into
  a green "reviewed" banner); **Yeniden aç** to edit again; **"AI taslağı"** revert.
- Backend: `edited_note`/`status`/`finalized_at` columns (lossless migration);
  `PATCH /notes/{id}` (409 if final), `POST /notes/{id}/finalize|reopen|revert`.

## 3. pytest suite (store + API)  (`dacbc87`, ADR-0017)
Reversed the "no unit tests" stance **for the deterministic non-ML layer**. Fast,
hermetic (temp DB, no models, <1 s) tests for the store + note/patient/edit/audio
endpoints. `make test`. The ML pipeline stays under the behavioral gate. (Building
this immediately caught a real test-isolation bug that had polluted the dev DB.)

## 4. Patient organization — Tier 1  (`dacbc87` backend + `88206f1` UX, REQ-137–140, ADR-0016)
- A lightweight `patients` table + nullable `notes.patient_id`. File a note under a
  patient; reuse a patient by name (case/space-insensitive); optional MRN.
- **UX (redesigned after your feedback):** the confusing free-text box is gone —
  now an explicit **"Hasta ata"** button → menu of existing patients + **"Yeni
  hasta"** dialog (Ad + Hasta No + **Kaydet**); assigned patient shows as a **chip**
  (remove ✕ / Değiştir). No accidental cross-note renames.
- Sidebar shows the patient per note + a **patient filter**. (Re)filing allowed
  even on a finalized note (filing is metadata, not content).

## 5. Search & filter — Tier 1  (`d5169cf`, REQ-141, ADR-0018)
Sidebar **"Notlarda ara…"** box (debounced) — case-insensitive match on title,

## 6. Export — Tier 1  (`6e038f2`, REQ-142)
A **"Dışa aktar"** menu on the note: **PDF olarak indir** (browser print-to-PDF via
a clean print stylesheet — no dependency, no server round-trip), **EHR için kopyala**
(markdown stripped to clean plain text for pasting into a hospital system),
Markdown kopyala, .md indir.

## 7. NEW: Audio-linked source transcript  (`db4c6fd`, REQ-143–146, ADR-0019)
**Your requested feature.** On a note, a **"Kaynak deşifre"** panel shows the source
transcript (speaker turns + timestamps); when the recording is available, an
embedded player lets you **click any turn to hear that exact moment** — so an
ambiguous/mis-transcribed passage can be checked against the original audio without
leaving the note. The active turn highlights as it plays.
- The structured turns are persisted with the note (`transcript_json`); the source
  audio is **copied** out of ephemeral job scratch into a durable, note-keyed,
  **git-ignored** store (`note_audio/<id>`) at note-persist time — so the link
  survives cleanup. Live-stream recordings are saved as a WAV for the same purpose.
- `GET /notes/{id}/audio` streams with **range support** (seek); deleting a note
  deletes its audio (PHI cleanup); path-safe (note-id validated, no traversal).
- **Graceful:** reused `out/` transcripts and older notes have no turns/audio — the
  panel hides or shows transcript-only; audio copy is best-effort and never fails a
  note.
